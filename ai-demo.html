1

📋 Crear encuesta de movilidad

Acceder a encuesta.html para crear un formulario web con 6 secciones: datos personales, desplazamiento habitual, valoración del entorno, satisfacción, teletrabajo y propuestas. El formulario exporta un CSV con 21 columnas.

Alternativa: Crear el CSV directamente en Excel/Google Sheets con las 21 columnas del formato.

2

📥 Importar respuestas CSV

En la app principal, pulsar ⚙️ Empresas → Importar CSV. El parser detecta automáticamente 30+ columnas con 15+ aliases (ej: "modo", "transporte", "medio" → todos mapean a modo principal). Soporta CSV con BOM, separadores coma/punto-coma, y encoding UTF-8/Windows-1252.

3

🏢 Configurar empresa y centro

Introducir: nombre, CIF, sector, empleados totales. En la pestaña Centro: nombre, dirección, código postal, ciudad, coordenadas GPS (latitud/longitud). Las coordenadas son clave para las APIs.

4

📊 Ejecutar diagnóstico y DAFO

Ir a Diagnóstico → Calcular. La IA genera automáticamente: reparto modal, indicadores de sostenibilidad, análisis DAFO (Fortalezas/Debilidades/Oportunidades/Amenazas), 15 medidas y 7 objetivos SMART.

5

🌐 Enriquecer con APIs reales

Si hay coordenadas, la app carga automáticamente: GBFS (estaciones BiciMAD), Overpass (paradas de bus/metro de OpenStreetMap), Nominatim (POIs del entorno). Sin API key para GBFS y Overpass.

6

🗺️ Ver mapa interactivo

La pestaña Mapa muestra: paradas TP (rojo), BiciMAD (verde/amarillo/rojo), centro (azul), y capa de isócronas con polígonos irregulares que simulan barreras urbanas.

7

📄 Generar informe y exportar

Pulsar 📥 Export → PDF. La herramienta genera un HTML de 170-180KB con 22 capítulos, tablas, gráficas, mapas y comentarios de la IA. WeasyPrint lo convierte a PDF de 71 páginas A4.

⚙️ Paso 6 — Medidas y objetivos SMART

15 medidas personalizadas y 7 objetivos con plazos

🎯 Generador automático (objetivos.js — 10KB) IA

Genera medidas y objetivos basándose en el DAFO. Cada objetivo sigue la metodología SMART:

  • Specífico — Qué se quiere conseguir
  • Medible — Indicador cuantificable
  • Alcanzable — Realista con los recursos
  • Relevante — Alineado con el PMST
  • Temporal — Plazo definido

🚲 GBFS — Bicicletas compartidas API

Sin API key. Feed público de sistemas de bicicletas compartidas en España (BiciMAD, Valenbisi, Bicing, etc.).

// Endpoint automático por ciudad GET https://gbfs.link/es/bicimad/stations.json // Datos por estación { "name": "59 - Paseo de la Habana", "bikesAvailable": 8, "slotsAvailable": 8, "latitude": 40.4455, "longitude": -3.6880 } // Detección automática por nombre de ciudad Madrid → BiciMAD | Barcelona → Bicing | Valencia → Valenbisi

🚌 Overpass API — Paradas de transporte público API

Sin API key. Consulta a OpenStreetMap para obtener paradas de bus, metro y tren en un radio de 800m.

// Consulta Overpass QL [out:json][timeout:15]; ( node["highway"="bus_stop"](40.44,-3.69,40.45,-3.68); node["public_transport"="stop_position"](40.44,-3.69,40.45,-3.68); ); out body 50; // Datos obtenidos por parada { "tags": { "name": "Paseo de la Habana", "ref": "14,27,40,147" }, "lat": 40.4462, "lon": -3.6885 }

🏙️ Nominatim — POIs del entorno API

Sin API key. Geocodificación inversa y búsqueda de puntos de interés: salud, educación, parking, hostelería.

// Búsqueda de POIs en radio 1km GET https://nominatim.openstreetmap.org/reverse?lat=40.4458&lon=-3.6888 // Categorías detectadas 🏥 Salud: hospital, clinic, pharmacy, doctors 🎓 Educación: school, university, kindergarten 🅿️ Parking: parking, bicycle_parking 🍽️ Hostelería: restaurant, cafe, bar

🗺️ Isócronas simuladas (sin ORS) MAPA

Cuando no hay API key de ORS, se generan isócronas simuladas pero realistas:

  • 8 ejes viales: Castellana (N), Bravo Murillo (NE), Paseo de la Habana (S)...
  • 2 barreras urbanas: Río Manzanares (-40%), vía de tren Chamartín (-25%)
  • Metro L9: extiende la isócrona norte un 25%
  • 48 puntos de resolución por polígono
  • 3 modos × 3 tiempos: coche/bici/pie × 10/15/30 min

🔢 Fórmula matemática

// Para cada uno de los 48 puntos del polígono angulo_base = (i / 48) × 360° factor_eje = ∏(1 + (eje.factor - 1) × peso) // 8 ejes factor_barrera = barrera.factor + (1 - barrera.factor) × diff // 2 barreras variacion = 1 + 0.15×sin(θ×0.1) + 0.1×cos(θ×0.23) + 0.08×sin(θ×0.37) radio = radio_max × factor_eje × factor_barrera × variacion // Conversión a coordenadas lat_punto = lat_centro + (radio / 111320) × cos(θ) lon_punto = lon_centro + (radio / (111320 × cos(lat))) × sin(θ) // Radio máximo por modo coche: 25 km/h × tiempo × 1000 / 60 bici: 14 km/h × tiempo × 1000 / 60 pie: 4.5 km/h × tiempo × 1000 / 60

🖼️ Cómo se capturan los mapas para el PDF

WeasyPrint no ejecuta JavaScript, así que los mapas Leaflet se capturan como imágenes:

  1. Se genera el HTML con mapas Leaflet
  2. Se abren en el navegador para que Leaflet renderice
  3. Se capturan con staticmap (OpenStreetMap tiles)
  4. Se convierten a JPEG comprimido (124-158KB)
  5. Se reemplazan los divs de mapa por <img>
  6. WeasyPrint genera el PDF con las imágenes incluidas
